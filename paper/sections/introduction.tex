\section{引言}

张量网络为高维数组的表示与计算提供了紧凑而有结构的形式，已广泛应用于量子多体计算、统计物理、图模型推断以及张量算法等方向\cite{orus2014practical}。
围绕张量网络收缩（TNC），已有研究主要关注三类问题：如何设计更优的收缩顺序，如何控制中间状态规模，以及如何通过中间状态压缩降低近似收缩的代价\cite{markov2008simulating,dumitrescu2018benchmarking,jakes2019carving,gray2021hyperoptimized,huang2021efficient,pan2020contracting,gray2024hyperoptimized,ma2024approximate}。
这些工作大多面向标量结果计算、局部可观测量计算或概率推断中的局部查询，而不是在保留开放腿的情况下显式生成完整输出张量\cite{orus2014practical,dechter1998bucket,pan2020contracting,heddes2026approximating}。
相比之下，一旦任务目标变为完整输出计算，计算与存储开销都会显著膨胀\cite{jakes2019carving}。
本文关注的正是这一类带开放腿张量网络的完整输出计算问题。

这一问题与通常的近似收缩存在根本区别：当完整输出本身就是任务目标时，算法不能将其替换为压缩表示，也不能只返回少量局部结果。
因此，本文关注的不是改变输出形式，而是在保持完整输出要求不变的前提下，降低生成该输出所需的计算开销。
若将端到端时间写为
\[
T_{\mathrm{total}} = T_{\mathrm{contract}} + T_{\mathrm{emit}},
\]
其中，$T_{\mathrm{contract}}$ 表示生成目标输出之前的内部计算开销，包括网络内部的收缩、合并以及相关中间状态处理；$T_{\mathrm{emit}}$ 表示将这些结果显式写入完整输出张量的开销。
由于完整输出本身必须被生成，$T_{\mathrm{emit}}$ 往往不可避免，因此方法设计的主要优化对象通常是 $T_{\mathrm{contract}}$。

基于这一观察，本文研究一种面向完整输出计算的近似收缩框架。
其基本思路是：先对输出索引空间进行分块，再在网络交互图上构造划分树，并沿树递归维护各个子网络的分隔状态；在整个过程中，输出边界始终保持显式，只对内部传播的分隔状态进行近似。
因此，本文近似的不是最终输出对象，而是“为了得到该输出而必须经过的内部中间状态”。
这也是本文与直接压缩输出端的方法，以及以标量收缩为目标的方法之间最本质的区别。

从表示上看，每个子树状态都可以理解为一个从输出侧到边界侧的接口矩阵。
本文在划分树上递归传播这一状态，并通过低秩因子化与合并后再压缩来抑制中间维度增长。
进一步地，针对内部节点合并时容易出现的接口膨胀，本文引入基于算子访问的隐式草图合并，不显式形成完整的合并接口，而是直接恢复其主导子空间。
因此，本文的核心并不是某一个孤立的局部技巧，而是将输出分块、划分树递归、分隔状态近似与草图驱动的合并压缩组织成一条统一的收缩流程。
在这条流程中，输出语义保持不变，近似只服务于降低内部收缩成本。

实验部分围绕三个问题展开：第一，\ours{} 在带开放腿张量网络收缩任务中能否形成清楚而可用的精度--时间折中；第二，这些收益是否确实来自收缩阶段，以及哪些机制在较难网络上最关键；第三，近似强度是否能够稳定控制误差。
结果表明，在代表性经典网络上，\ours{} 将相对误差控制在 $4.81\times 10^{-3}$ 以内，同时相对直接 TNC 获得 $13.98\times$ 到 $84.05\times$ 的加速；而固定秩基线虽然更快，却带来 $0.65$ 到 $0.98$ 的大误差。
进一步分析表明，端到端收益几乎完全来自收缩阶段；隐式草图合并是更稳定也更核心的效率来源，输出分块同样具有实质作用。
近似强度扫描实验则显示，近似强度可以作为控制误差的有效调节旋钮。

本文的贡献可以概括为三点：
\begin{itemize}
    \item 针对带开放腿张量网络的完整输出计算任务，本文将近似的位置从输出端移到内部收缩过程，关注在保持完整输出要求不变的前提下降低内部计算代价。
    \item 本文提出 \ours{} 框架：通过输出分块与划分树递归维护分隔状态，利用自适应低秩压缩控制状态规模，并在内部合并处引入隐式草图合并，以避免显式形成膨胀接口。
    \item 本文在 Ring-8、Tree-8、Grid $3\times 3$ 与 Random-8 上系统验证了该方法，结果表明 \ours{} 能够形成清楚的精度--时间折中，并揭示了其主要收益来源与关键机制。
\end{itemize}
