\section{引言}

张量网络为高维数组的表示与计算提供了紧凑而有结构的形式，已经广泛用于量子多体计算、统计物理、图模型推断以及张量算法等方向\cite{orus2014practical}。
围绕张量网络收缩，已有工作主要关心三类问题：如何设计更好的收缩顺序，如何控制中间状态规模，以及如何通过中间状态压缩来降低近似收缩的代价\cite{markov2008simulating,dumitrescu2018benchmarking,jakes2019carving,gray2021hyperoptimized,huang2021efficient,pan2020contracting,gray2024hyperoptimized,ma2024approximate}。
现有张量网络收缩研究大多围绕标量收缩、局部可观测量计算或概率推断中的局部查询展开，而非显式生成保留开放腿后的完整输出张量\cite{orus2014practical,dechter1998bucket,pan2020contracting,heddes2026approximating}。
相比之下，一旦要求对若干开放腿对应的全部配置进行物化，输出空间和中间收缩状态都会迅速膨胀，从而带来显著的计算与存储开销\cite{jakes2019carving}。
本文关注的正是这一类带开放腿张量网络的完整稠密输出计算问题。

这一问题与通常的近似收缩有一个根本差别：当完整输出本身就是任务目标时，算法不能将其替换为压缩表示，也不能仅返回少量局部结果。
因此，我们关注的问题不是改变输出形式，而是在保持完整输出要求不变的前提下，降低生成这一输出的计算开销。
若把端到端时间写成
\[
T_{\mathrm{total}} = T_{\mathrm{contract}} + T_{\mathrm{emit}},
\]
我们将 $T_{\mathrm{contract}}$ 理解为生成目标输出之前的内部计算开销，即网络内部的收缩、合并及相关中间状态处理；将 $T_{\mathrm{emit}}$ 理解为把这些结果显式物化为完整输出张量的开销。
由于完整输出本身必须被生成，$T_{\mathrm{emit}}$ 往往不可回避，因此方法设计的主要优化对象通常是 $T_{\mathrm{contract}}$。

基于这一观察，本文研究一种面向完整物化的近似收缩框架。
基本思路是：在网络交互图上构造一棵划分树，并沿树递归维护各个子网络的局部接口；在整个过程中，输出边界始终保持显式，只对内部接口做近似。
于是，本文近似的不是最终输出对象，而是“为了得到该输出而必须经过的内部中间状态”。
这也是本文与直接压缩输出端的方法，以及以标量收缩为目标的方法之间最本质的区别。

从表示上看，每个子树状态都可以理解为一个从输出侧到边界侧的接口矩阵；本文用低秩因子化在树上递归传播这一状态，并在每次合并后重新压缩，以抑制中间维度增长。
进一步地，针对内部节点合并时容易出现的秩乘积膨胀，本文在 \method\ 中引入基于算子访问的随机化 sketch 压缩，不显式形成完整的合并接口，而是直接恢复其主导子空间。
因此，本文的核心并不是某一个孤立的局部技巧，而是把输出分块、划分树递归、分隔状态近似与 sketch 驱动的合并压缩组织成一条统一的完整收缩流程。
在这条流程里，输出语义保持不变，近似只服务于降低内部收缩成本。

实验围绕两组问题展开。
第一，相对精确完整收缩，\method\ 在尺度扩展过程中能否保持清楚的误差--时间折中，并在更大规模上继续保持可行；
第二，固定秩、缓存、选择性近似和 implicit merge sketch 等内部设计到底分别起什么作用。
结果表明：在 exact 可得区间内，\method\ 形成了清楚的误差--时间折中，并给出相对 exact 的数量级加速；阶段分解显示端到端收益主要来自 contraction side。机制层面上，fixed-rank 更接近“更快但更不准”的端点，implicit merge sketch 是 harder cases 上更稳健的效率来源，blockwise output organization 在更复杂 workload 上更重要但具有 workload dependence，而 cache reuse 在当前实现下不支持稳定正收益的统一结论。
在跨尺度评估中，easy regime 行为稳定，但 medium 区间已经出现明显 boundary-like behavior，boundary 区间则出现大面积 timeout/failure。因而，\textsf{ASTNC-L2} 仅被用作主实验中的 pragmatic default preset，而不是经验上全局最优工作点；更大规模下的推荐也应采用 case-dependent 口径。

本文的贡献可以概括为三点：
\begin{itemize}
    \item 提出一个面向开放腿张量网络完整物化的近似收缩设定，将优化重点从“近似收缩值”转向“在保持输出边界语义不变的前提下降低内部收缩代价”。
    \item 提出一种基于划分树递归的分隔状态框架，并在 \method\ 中引入基于算子访问的 implicit merge sketch，以降低内部合并阶段的大接口压缩开销。
    \item 通过 exact-reference comparison、阶段时间分解、尺度 regime 评估与内部消融说明：\method\ 在完整物化任务中形成可解释的误差--时间折中，主要收益来自 contraction side；implicit sketch 在 harder cases 上更稳健，blockwise 具有 workload dependence，cache reuse 在当前实现下不支持 universal positive claim。
\end{itemize}
