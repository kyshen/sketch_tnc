\section{实验}

本节围绕带开放腿张量网络的完整稠密物化任务，对 \method\ 的误差--时间折中、收益来源、内部机制作用、近似强度敏感性以及跨拓扑跨尺度行为进行系统分析。
与通常只关注标量收缩值的近似收缩设定不同，本文要求显式生成完整输出张量，因此实验更关注在保持输出语义不变的前提下，内部收缩代价是否能够被有效降低。
当前阶段仅汇报主文表格结果。

\subsection{实验设置与评测协议}

实验覆盖 chain、ring、tree、grid2d 和 random\_connected 五类拓扑，并按 easy、medium、boundary 三个规模区间组织。
其中 easy 区间用于建立 exact 可得条件下的主结果与机制分析；medium 与 boundary 区间用于观察尺度扩大后的边界行为。

评测指标分为三类：
精度指标（rel\_error、RMSE、NMSE、NMSE\_dB）、时间指标（contraction/emit/total 及相对 exact 的 speedup）、内部统计（如 peak rank、compressed merge 数、implicit sketch 触发统计、cache 统计等）。
除特别说明外，\textsf{ASTNC-L2} 在主实验中作为默认预设，仅代表务实的中间配置，而非经验上全局最优统一工作点。

\subsection{主结果：端到端误差--时间折中}

主结果如表~\ref{tab:main-results-casewise} 与表~\ref{tab:main-results-summary} 所示。
在核心 case（ring/tree/grid2d/random）上，exact 始终作为零误差锚点，但端到端时间最高；fixed-rank baseline 往往最快，但误差在非平凡拓扑上明显恶化；\textsf{ASTNC-L2} 在多数核心 case 上保持低误差并提供数量级加速，形成可用的误差--时间折中。

特别是在 $3\times 3$ grid 上，\textsf{ASTNC-L2} 相对 exact 达到 84.05$\times$ 加速，rel\_error 仍维持在 $4.08\times 10^{-3}$；对比 fixed-rank baseline，尽管后者更快，但误差已接近 $10^{0}$ 量级。
跨 case 汇总中，\textsf{ASTNC-L2} 的平均 speedup 为 50.95$\times$（几何均值 42.53$\times$），平均误差显著低于 fixed-rank baseline。

\subsection{收益来自哪里：contraction-side gain analysis}

阶段时间分解见表~\ref{tab:gain-breakdown-stage}，收益汇总见表~\ref{tab:gain-breakdown-summary}。
在全部核心 case 上，$t_{\mathrm{contract}}/t_{\mathrm{total}}$ 接近 1，而 $t_{\mathrm{emit}}/t_{\mathrm{total}}$ 始终为小尾项，说明端到端时间由 contraction 阶段主导。

对 \textsf{ASTNC-L2} 而言，平均 contract speedup（52.07$\times$）与平均 total speedup（52.03$\times$）几乎一致，且 gain\_share\_contract 约为 1。
因此，\method\ 的主要收益来自 contraction side，而非 emit/writeback 端。
个别 gain-share 的轻微扰动可由 emit 项量级极小导致的测量波动解释。

\subsection{核心机制消融}

核心消融结果见表~\ref{tab:ablation-core}。
总体上，\method\ 的收益并非单一技巧驱动，而是多个机制组合形成。

第一，implicit merge sketch 在 harder case 上更稳健且更关键：在 main\_grid2d\_3x3 上去除 implicit 后，时间相对 \textsf{ASTNC-L2} 恶化到 6.97$\times$，而误差基本不变。
第二，blockwise output organization 具有明显 workload dependence：在 main\_ring\_8 上 no-blockwise 反而更快，但在 grid2d 与 random 上明显变慢并伴随更高峰值秩。
第三，cache reuse 在当前实现下不支持 universal positive claim：no-cache 在不同 case 上既有变快也有变慢，表现出实现开销敏感和 workload 依赖。

据此，正文主叙事聚焦两点：implicit merge sketch 是 harder cases 上更稳健的效率来源；blockwise 在复杂收缩结构中更重要但非无条件增益。

\subsection{近似强度敏感性}

强度敏感性结果见表~\ref{tab:strength-sensitivity}。
在核心 case 上，误差随强度从 L1$\rightarrow$L2$\rightarrow$L3 单调恶化；但运行时间不呈单调规律。
这意味着“更强近似必然更快”在当前数据上并不成立。

例如在 main\_grid2d\_3x3 上，\textsf{ASTNC-L1} 同时快于且准于 \textsf{ASTNC-L2}；\textsf{ASTNC-L3} 虽有部分时间回收，但误差显著上升。
在 main\_random\_8 上，L1 与 L2 都接近 near-exact，而 L3 主要带来误差恶化且无稳定额外收益。
因此，\textsf{ASTNC-L2} 在全文中仅应作为 pragmatic default preset，而非统一最优工作点。

\subsection{跨拓扑跨尺度行为与边界区间}

跨尺度与区间归类见表~\ref{tab:regime-inventory} 与表~\ref{tab:regime-summary}。
easy 区间（5/5 case）全部完成且 exact 锚点可得，3.2--3.5 的主结论在该区间内稳定成立。

更关键的是，boundary-like behavior 在 medium 区间已经出现：平均 completed fraction 仅 0.075，平均 timeout rate 达 0.925，exact 锚点全部缺失。
在 nominal boundary 区间，完成率降至 0，timeout rate 为 1.000。
这说明当前实现并未经历较长的平滑过渡区，而是从 fully stable easy regime 较快进入 boundary regime。

因此，timeout/failure 不是应丢弃的坏数据，而是有意义的边界信号；更大尺度下的方法推荐必须以 case/regime 为条件，而不能沿用 easy 区间中的统一口径。

综合 3.2--3.6 的结果，本文支持如下结论：在 exact 可得区间内，\method\ 形成了清楚的误差--时间折中；其主要收益来自 contraction side；implicit merge sketch 在 harder cases 上更稳健，blockwise 在复杂 workload 上更重要但具 workload dependence；cache reuse 的收益在当前实现下不稳定；\textsf{ASTNC-L2} 仅是务实默认预设而非统一最优；规模扩大后当前实现较快进入 boundary regime。
