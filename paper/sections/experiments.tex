\section{实验}

本节围绕三个问题组织实验：第一，\method\ 在 TNC 任务中是否形成了可用的精度--时间折中；第二，这些收益是否真的来自内部收缩，而不是输出写回；第三，近似强度与关键机制分别起什么作用。

\subsection{实验设置}

我们主要报告四类有代表性的经典网络：ring~\cite{bondy2008graph}、tree~\cite{diestel2017graph}、grid2d~\cite{levin2007trg,orus2014practical} 与随机网络~\cite{erdos1959random,gilbert1959random}，具体为：Ring-8、Tree-8、Grid 3 $\times$ 3 与 Random-8（8 为节点数，3 $\times$ 3代表二维网格）；每个节点都带 1 条开放腿，物理维统一取 $3$，bond 维统一取 4。\textsf{\method-L1/L2/L3}对应不同的近似强度，除特别说明外，\textsf{\method} 指中等强度预设。对比方法包括精确收缩 exact-TNC、固定秩基线与 \method。固定秩基线统一设置秩为4，目的是提供一个明确的“小秩快速近似”参照。其他关键细节（网络构造、容忍度$\tau$、分块策略等）见附录~\Cref{app:exp-details}。

\subsection{主结果}

\Cref{tab:main-core} 给出了四个网络上的比较。可以看到，固定秩基线的共同特征是极快，但在 ring、tree、grid 与随机网络上都出现了明显失真：其相对误差达到 $0.65$ 到 $0.98$。相比之下，默认 \method 配置在四个网络上都把相对误差压在 $4.81\times 10^{-3}$ 以内，同时相对 exact-TNC 仍获得 $13.98\times$ 到 $84.05\times$ 的加速。最有区分度的结果来自 Grid 3 $\times$ 3：exact-TNC 需要 $37.20$ 秒，而 \method 将总时间降到 $0.4426$ 秒，相对误差仅为 $4.08\times 10^{-3}$。在这四个网络上，\method 的几何平均加速为 $42.53\times$，平均相对误差为 $2.68\times 10^{-3}$。这说明 \method\ 在保持较高精度的同时，能够实现显著的时间加速。

\subsection{机制分析}

本节探讨所提出方法的收益来自哪里。阶段分解给出的答案非常明确：在四个主结果网络上，默认 \method 配置的平均 $t_{\mathrm{contract}}/t_{\mathrm{total}}$ 为 $0.9992$；其平均收缩加速比与总加速比分别为 $52.07\times$ 与 $52.03\times$，几乎完全重合。这说明 \method 的端到端收益几乎完全来自收缩阶段，而不是写入阶段的偶然波动。以 Grid 3 $\times$ 3 为例，收缩加速为 $84.36\times$，总加速为 $84.33\times$，二者在数值上几乎没有区别。

在机制层面，以 Grid 3 $\times$ 3 这一最具区分度的网络为例\Cref{tab:grid-evidence}。去掉 implicit merge sketch 后，误差几乎不变，但总时间恶化到默认设置的 $6.97\times$；这说明隐式草图融合降低内部合并成本且不牺牲精度。去掉输出分块机制后，总时间变为 $2.40\times$，同时误差从 $4.08\times 10^{-3}$ 上升到 $1.06\times 10^{-2}$。综合这些结果，本文在机制上的结论是：隐式草图融合是核心、稳定的效率来源，输出分块也具有实质作用。

\subsection{近似强度调控}

本节探究近似强度 $\tau$ 是否能稳定控制误差。从 \textsf{L1} 到 \textsf{L2} 再到 \textsf{L3}，误差呈现出清楚的递增关系。以 Ring-8 为例，相对误差从 $<10^{-12}$ 上升到 $1.83\times 10^{-3}$，再到 $1.55\times 10^{-2}$；在 Grid 3 $\times$ 3 上，则从 $<10^{-12}$ 上升到 $4.08\times 10^{-3}$，再到 $1.81\times 10^{-2}$；Random-8 上 \textsf{L1}/\textsf{L2} 近乎无误差，而 \textsf{L3} 上升到 $1.26\times 10^{-2}$。因此，近似强度首先是 \method\ 控制误差的旋钮。
