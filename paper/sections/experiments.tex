\section{实验}

本节围绕三个问题组织实验：第一，\ours{} 在带开放腿张量网络的完整输出计算任务中，是否形成了清楚而可用的精度--时间折中；第二，这些收益是否确实来自内部收缩，以及隐式草图合并、缓存复用与输出分块分别起什么作用；第三，近似强度是否能够作为稳定的误差调控旋钮。

\subsection{实验设置}

我们在带开放腿张量网络的完整输出计算任务上评估 \ours{}。
主结果覆盖四类有代表性的网络：Ring-8、Tree-8、Grid $3\times 3$ 与 Random-8。
除网络拓扑外，其余张量规模统一：每个节点带 1 条开放腿，物理维为 $3$，键维为 $4$。

除特别说明外，正文中的 \ours{} 均指默认工作点 \textsf{ASTNC-L2}。
该默认工作点同时启用三项核心机制：输出分块、隐式草图合并与块间缓存复用。
其中，输出分块沿前两个开放标签逐块生成完整输出，且每个分块轴上的局部片段长度为 $1$。
固定秩基线统一取 rank $=4$，用于提供一个“小秩但快速”的近似参照。

对比方法包括精确基线 \exact{}、固定秩基线 \fixrank{} 与自适应版本 \ours{}。
为保证比较公平，所有方法共享同一张量网络实例、同一开放标签顺序、同一组输出块以及相同的块级写回接口；差别仅在内部收缩与压缩的组织方式。
因此，正文中的速度与误差差异可以直接归因于内部计算策略，而非任务定义或输出接口差异。

可复现实验所需的数值级配置——包括网络生成规则、随机种子、三档强度预设、默认工作点与论文记号的对应关系，以及统一的计时与汇总口径——集中放在附录~\Cref{app:exp-details}。

\subsection{主结果}

\Cref{tab:main-core} 给出了四个典型网络上的主结果。
可以看到，固定秩基线虽然很快，但在四个网络上都产生了明显失真：其相对误差介于 $0.65$ 到 $0.98$ 之间。
相比之下，默认 \ours{} 配置在四个网络上都将相对误差控制在 $4.81\times 10^{-3}$ 以内，同时相对 \exact{} 仍获得了 $8.97\times$ 到 $88.45\times$ 的加速。
最具区分度的结果来自 Grid $3\times 3$：\exact{} 需要 $47.25$ 秒，而 \ours{} 将总时间降到 $0.5342$ 秒，相对误差仅为 $4.08\times 10^{-3}$。
在四个主结果网络上，\ours{} 的几何平均加速达到 $35.08\times$，平均相对误差为 $2.68\times 10^{-3}$。
这表明 \ours{} 在保持较高精度的同时，能够显著降低完整输出计算所需总时间。

\subsection{收缩主导与机制作用}

首先，我们考察端到端收益究竟来自哪里。
由 \Cref{tab:stage-breakdown-core} 可见，在四个主结果网络上，默认 \ours{} 配置的平均 $t_{\mathrm{contract}}/t_{\mathrm{total}}$ 达到 $0.9993$；其平均收缩加速比与总加速比分别为 $46.25\times$ 与 $46.23\times$，几乎完全一致。
这说明 \ours{} 的主要收益确实来自内部收缩阶段，而不是输出写回。
以 Grid $3\times 3$ 为例，收缩加速为 $88.48\times$，总加速为 $88.45\times$，二者在数值上几乎重合。

接着，我们在 Grid $3\times 3$ 这一最具区分度的网络上隔离关键机制。
结果如 \Cref{tab:grid-mechanism} 所示。
去掉隐式草图合并后，误差几乎不变，但总时间恶化到默认设置的 $2.21\times$；这说明隐式草图合并是稳定且核心的效率来源。
去掉缓存复用后，误差同样几乎不变，但总时间上升为默认设置的 $1.23\times$，表明块间状态复用能够带来额外但稳定的系统收益。
相比之下，去掉输出分块后，总时间并未显著变差，但误差从 $4.08\times 10^{-3}$ 上升到 $1.06\times 10^{-2}$，同时内部峰值秩由 $64$ 上升到 $222$。
这说明输出分块在当前小规模设置下更像是在塑造一个更优的精度--效率工作点，而不是一个单调的纯加速开关。

\subsection{近似强度与连续前沿}

我们接着考察近似强度是否能够作为稳定的误差调控旋钮。
\Cref{tab:strength-sensitivity-core} 给出了离散的三档强度扫描结果。
可以看到，从 \textsf{L1} 到 \textsf{L2} 再到 \textsf{L3}，误差整体上呈现清楚的上升趋势。
以 Grid $3\times 3$ 为例，相对误差从近乎机器精度上升到 $4.08\times 10^{-3}$，再增至 $1.81\times 10^{-2}$；Tree-8 上则从近乎零误差上升到 $4.81\times 10^{-3}$，再达到 $4.86\times 10^{-2}$。
四个网络上的平均相对误差分别为近乎零、$3.59\times 10^{-3}$ 与 $2.37\times 10^{-2}$。
这说明近似强度首先是一个有效的误差调控旋钮：更弱的近似更接近精确结果，而更强的近似则带来更大的误差。
